POO Java

es un paradigma de programación que utiliza objetos para organizar y estructurar el código. Java es un lenguaje de programación que sigue fuertemente el paradigma de POO.

programación concurrente en java

se refiere a la capacidad de ejecutar múltiples tareas en paralelo, permitiendo que partes de un programa se ejecuten simultáneamente. Java proporciona varias características y bibliotecas para facilitar la programación concurrente


Hilos (Threads): Un hilo es una secuencia de ejecución independiente dentro de un programa. Java permite la creación y gestión de hilos utilizando la clase Thread o implementando la interfaz Runnable.


Ejemplo:
public class EjemploHilo {
// Clase que extiende Thread
static class MiHilo extends Thread {
public void run() {
for (int i = 0; i < 5; i++) {
try {
Thread.sleep(1000); // Simular una tarea que lleva tiempo
} catch (InterruptedException e) {
e.printStackTrace();
}
System.out.println("Número: " + i);
}
}
}
// Clase que implementa Runnable
static class MiRunnable implements Runnable {
public void run() {
for (char letra : new char[]{'a', 'b', 'c'}) {
try {
Thread.sleep(500);
} catch (InterruptedException e) {
e.printStackTrace();
}
System.out.println("Letra: " + letra);
}
}
}
public static void main(String[] args) {
// Crear e iniciar un hilo que extiende Thread
MiHilo miHilo = new MiHilo();
miHilo.start();
// Crear e iniciar un hilo que implementa Runnable
Thread miHiloRunnable = new Thread(new MiRunnable());
miHiloRunnable.start();
// Realizar otras tareas en el hilo principal
for (int i = 0; i < 3; i++) {
System.out.println("Tarea en el hilo principal");
try {
Thread.sleep(1000);
} catch (InterruptedException e) {
e.printStackTrace();
}
}
System.out.println("Programa principal finalizado.");
}
}


Métodos sincronizados: son aquellos que han sido marcados con la palabra clave synchronized. Esta característica permite que solo un hilo ejecute el método sincronizado a la vez. La sincronización es esencial para evitar condiciones de carrera y mantener la consistencia de datos compartidos entre varios hilos.


Ejemplo:
public class EjemploSincronizacionBloque {
private int contador = 0;
private Object cerrojo = new Object();
public void incrementar() {
// Bloque sincronizado
synchronized (cerrojo) {
contador++;
}
}
}


Candados implícitos: uso del cerrojo intrínseco asociado a cada objeto para implementar la sincronización en métodos o bloques de código. Cada objeto en Java tiene un cerrojo asociado internamente, y este cerrojo puede ser utilizado para lograr la sincronización.


Ejemplo:
public class EjemploSincronizacionBloque {
private int contador = 0;
public void incrementar() {
// Bloque sincronizado utilizando el cerrojo intrínseco
synchronized (this) {
contador++;
}
}
}


Volatile:se utiliza para indicar que una variable puede ser modificada por múltiples hilos simultáneamente. Al declarar una variable como volatile, se garantiza que las lecturas y escrituras de esa variable son atómicas y que no se realizan optimizaciones de caché que podrían interferir con la operación de múltiples hilos.


Ejemplo:
public class EjemploVolatile {
private volatile boolean bandera = false;
public void cambiarBandera() {
bandera = true;
}
public void imprimirMensaje() {
while (!bandera) {
System.out.println("Esperando a que la bandera cambie...");
}
System.out.println("La bandera ha cambiado.");
}
public static void main(String[] args) {
EjemploVolatile ejemplo = new EjemploVolatile();
// Hilo que cambia la bandera
new Thread(() -> {
try {
Thread.sleep(2000); // Simula alguna operación que toma tiempo
} catch (InterruptedException e) {
e.printStackTrace();
}
ejemplo.cambiarBandera();
}).start();
// Hilo que imprime el mensaje
new Thread(() -> {
ejemplo.imprimirMensaje();
}).start();
}
}


Interferencia entre hilos contención: Cuando múltiples hilos comparten recursos o datos, puede producirse una situación conocida como "contención" o "interferencia entre hilos". La contención ocurre cuando dos o más hilos intentan acceder o modificar un recurso compartido al mismo tiempo. Esta situación puede llevar a resultados inesperados y errores en el programa.


Ejemplo:
public class EjemploInterferencia {
static class Contador {
private int valor = 0;
// Método sin sincronización
void incrementar() {
valor++;
}
// Método sincronizado para evitar interferencia y contención
synchronized void incrementarSincronizado() {
valor++;
}
int getValor() {
return valor;
}
}
static class MiHilo implements Runnable {
private Contador contador;
MiHilo(Contador contador) {
this.contador = contador;
}
public void run() {
for (int i = 0; i < 10000; i++) {
// Descomentar la línea correspondiente para ver la interferencia o contención
// contador.incrementar(); // Interferencia y contención
contador.incrementarSincronizado(); // Sin interferencia ni contención
}
}
}
public static void main(String[] args) throws InterruptedException {
Contador contador = new Contador();
// Crear dos hilos que incrementan el contador sin sincronización
Thread hilo1 = new Thread(new MiHilo(contador));
Thread hilo2 = new Thread(new MiHilo(contador));
hilo1.start();
hilo2.start();
hilo1.join();
hilo2.join();
// Imprimir el resultado final
System.out.println("Valor final del contador: " + contador.getValor());
}
}

archivo = MainClas.java
package concurrenciaCorte3;
/**
*
* @author estudiante
*/
public class MainClass {
public void run(){
System.out.println("Ejecutando hilo"+Thread.currentThread().getId());
}
public static void main(String[] args)
{
Account myAccount = new Account(50000000);
//1. crar 2 hilos para las dos operaciones de la cuanta
Thread hiloRetiro = new Thread( ()-> myAccount.withdraw(200000));
Thread hiloDeposito = new Thread( ()-> myAccount.deposit(50000));
Thread[] hiloReTiro = new Thread[8];
for (int i = 0; i < hiloReTiro.length; i++) {
hiloReTiro[i] = new Thread( ()-> myAccount.withdraw(1000000));
hiloReTiro[i].start();
}
//2. inicializar los hilos
hiloRetiro.start();
hiloDeposito.start();
//3. unir los hilos a este hilo principal (main)
try {
hiloDeposito.join();
hiloRetiro.join();
for(Thread t: hiloReTiro){
t.join();
System.out.println("hilo con id: " + t.getId());
}
} catch (InterruptedException e) {
e.printStackTrace();
}
System.out.println("balance final:"+myAccount.getBalance());
//operaciones secuenciales:
myAccount.deposit(500000);
myAccount.withdraw(40000);
}
}
