POO en Python

La Programación Orientada a Objetos (POO) es un paradigma de programación que se basa en el concepto de "objetos", que pueden contener datos en forma de campos, a menudo conocidos como atributos, y código, en forma de procedimientos, a menudo conocidos como métodos. Python es un lenguaje que admite la programación orientada a objetos y proporciona muchas características para trabajar con ella

Clases

es un plano o plantilla que define un tipo de objeto. Una clase encapsula datos (atributos) y funciones (métodos) que operan en esos datos. Puedes pensar en una clase como un prototipo para la creación de objetos.

Ejemplo:
class Persona:
def __init__(self, nombre, edad):
self.nombre = nombre
self.edad = edad
def saludar(self):
print(f"Hola, soy {self.nombre} y tengo {self.edad} años.

Objetos

es una instancia específica de una clase. Puedes crear múltiples objetos a partir de la misma clase, y cada objeto tendrá sus propios valores para los atributos

Ejemplo:
persona1 = Persona("Alice", 30)
persona2 = Persona("Bob", 25)
print(persona1.nombre)
persona2.saludar()

Atributos y Métodos

Los atributos son variables asociadas con la clase u objeto, y los métodos son funciones asociadas

Ejemplo:
class Estudiante(Persona):
def __init__(self, nombre, edad, curso):
super().__init__(nombre, edad)
self.curso = curso
def estudiar(self):
print(f"{self.nombre} está estudiando en el curso {self.curso}.")

Encapsulamiento

En Python, la encapsulación se logra utilizando convenciones de nomenclatura y, en algunos casos, mediante el uso de atributos y métodos privados.

Ejemplo:
class CuentaBancaria:
def __init__(self, saldo):
self.__saldo = saldo # __saldo es un atributo privado
def obtener_saldo(self):
return self.__saldo

Herencia

La herencia permite que una clase herede atributos y métodos de otra clase. En Python, una clase hereda de otra utilizando paréntesis después del nombre de la clase

Ejemplo:
# Acceder a atributos y llamar a métodos
print(persona1.nombre)
persona2.saludar()

Polimorfismo

El polimorfismo permite que objetos de diferentes clases sean tratados como objetos de una misma clase base.

Ejemplo:
def imprimir_detalle(objeto):
print(objeto.detalle())
# Polimorfismo en acción
imprimir_detalle(persona1)
imprimir_detalle(estudiante1)

DRY = Don’t Repeat Yourself

El procesamiento de listas es una parte esencial de la programación en Python, y hay varias formas de trabajar con listas

Ejemplo:
# Enfoque no DRY
def operacion_a():
# lógica específica de operación_a
pass
def operacion_b():
# lógica específica de operación_b
pass

Sería preferible definir una función o clase que encapsule la lógica común


# Enfoque DRY
def operacion_comun():
# lógica común a operación_a y operacion_b
pass
def operacion_a():
operacion_comun()
# lógica específica de operación_a
pass
def operacion_b():
operacion_comun()
# lógica específica de operación_b
pass

Patrones de diseño

Los patrones de diseño son soluciones generales y reutilizables para problemas recurrentes en el diseño de software. Estos patrones proporcionan un enfoque probado y estructurado para resolver ciertos problemas y mejorar la eficiencia del diseño del software. No son fragmentos de código o bibliotecas que puedes simplemente incluir en tu aplicación, sino más bien pautas y conceptos para abordar ciertos tipos de problemas


Singleton (Singleton): Garantiza que una clase tenga solo una instancia y proporciona un punto de acceso global a esa instancia.


Ejemplo:
class Singleton:
_instance = None
def __new__(cls):
if cls._instance is None:
cls._instance = super(Singleton, cls).__new__(cls)
return cls._instance
# Uso del Singleton
singleton_1 = Singleton()
singleton_2 = Singleton()
print(singleton_1 is singleton_2)


Factory Method (Método de fábrica): Define una interfaz para crear un objeto, pero deja que las subclases alteren el tipo de objetos que se crearán.


Ejemplo:
from abc import ABC, abstractmethod
# Interfaz para crear productos
class Creator(ABC):
@abstractmethod
def factory_method(self):
pass
# Clase concreta que implementa la interfaz
class ConcreteCreator(Creator):
def factory_method(self):
return ConcreteProduct()
# Interfaz para los productos
class Product(ABC):
@abstractmethod
def operation(self):
pass
# Clase concreta que implementa un producto
class ConcreteProduct(Product):
def operation(self):
return "Producto concreto"
# Uso del Factory Method
creator = ConcreteCreator()
product = creator.factory_method()
print(product.operation())


Abstract Factory (Fábrica abstracta): Proporciona una interfaz para crear familias de objetos relacionados o dependientes sin especificar sus clases concretas.


Ejemplo:
from abc import ABC, abstractmethod # Interfaz para la fábrica abstracta
class AbstractFactory(ABC):
@abstractmethod
def create_product_a(self):
pass
@abstractmethod
def create_product_b(self):
pass
# Clases concretas que implementan la fábrica abstracta
class ConcreteFactory1(AbstractFactory):
def create_product_a(self):
return ConcreteProductA1()
def create_product_b(self):
return ConcreteProductB1()
class ConcreteFactory2(AbstractFactory):
def create_product_a(self):
return ConcreteProductA2()
def create_product_b(self):
return ConcreteProductB2()
# Interfaz para los productos A
class AbstractProductA(ABC):
@abstractmethod
def operation_a(self):
pass
# Clases concretas que implementan productos A
class ConcreteProductA1(AbstractProductA):
def operation_a(self):
return "Operación A para Producto A1"
class ConcreteProductA2(AbstractProductA):
def operation_a(self):
return "Operación A para Producto A2"
# Interfaz para los productos B
class AbstractProductB(ABC):
@abstractmethod
def operation_b(self):
pass
# Clases concretas que implementan productos B
class ConcreteProductB1(AbstractProductB):
def operation_b(self):
return "Operación B para Producto B1"
class ConcreteProductB2(AbstractProductB):
def operation_b(self):
return "Operación B para Producto B2"
# Uso de la Abstract Factory
factory1 = ConcreteFactory1()
product_a1 = factory1.create_product_a()
product_b1 = factory1.create_product_b()
print(product_a1.operation_a()) # Salida: "Operación A para Producto A1"
print(product_b1.operation_b()) # Salida: "Operación B para Producto B1"
factory2 = ConcreteFactory2()
product_a2 = factory2.create_product_a()
product_b2 = factory2.create_product_b()
print(product_a2.operation_a()) # Salida: "Operación A para Producto A2"
print(product_b2.operation_b()) # Salida: "Operación B para Producto B2"


Adapter (Adaptador):Permite que la interfaz de una clase existente se utilice como si fuera otra interfaz.


Ejemplo:
# Interfaz Target class Target:
def request(self):
pass
# Clase Adaptee (la clase que tiene la interfaz incompatible)
class Adaptee:
def specific_request(self):
return "Solicitud específica"
# Adaptador que conecta la interfaz Target con la interfaz Adaptee
class Adapter(Target):
def __init__(self, adaptee):
self.adaptee = adaptee
def request(self):
return f"Adapter: {self.adaptee.specific_request()}"
# Uso del Adapter
adaptee = Adaptee()
adapter = Adapter(adaptee)
result = adapter.request()
print(result) # Salida: "Adapter: Solicitud específica"


Observer (Observador): Define una dependencia uno a muchos entre objetos para que cuando uno de los objetos cambie su estado, todos sus dependientes sean notificados y actualizados automáticamente.


Ejemplo: from abc import ABC, abstractmethod # Interfaz Observer
class Observer(ABC):
@abstractmethod
def update(self, message):
pass
# Clase ConcreteObserver
class ConcreteObserver(Observer):
def __init__(self, name):
self.name = name
def update(self, message):
print(f"{self.name} recibió el mensaje: {message}")
# Clase Subject (Sujeto)
class Subject:
def __init__(self):
self._observers = []
def add_observer(self, observer):
self._observers.append(observer)
def remove_observer(self, observer):
self._observers.remove(observer)
def notify_observers(self, message):
for observer in self._observers:
observer.update(message)
# Uso del Observer
subject = Subject()
observer1 = ConcreteObserver("Observer 1")
observer2 = ConcreteObserver("Observer 2")
subject.add_observer(observer1)
subject.add_observer(observer2)
subject.notify_observers("¡Estado actualizado!")


Strategy (Estrategia): Define una familia de algoritmos, encapsula cada uno y los hace intercambiables. Permite que el algoritmo varíe independientemente de los clientes que lo utilizan.


Ejemplo:
from abc import ABC, abstractmethod # Interfaz para la estrategia
class Strategy(ABC):
@abstractmethod
def execute(self):
pass
# Estrategia concreta 1
class ConcreteStrategy1(Strategy):
def execute(self):
return "Estrategia concreta 1"
# Estrategia concreta 2
class ConcreteStrategy2(Strategy):
def execute(self):
return "Estrategia concreta 2"
# Contexto que utiliza una estrategia
class Context:
def __init__(self, strategy):
self._strategy = strategy
def set_strategy(self, strategy):
self._strategy = strategy
def execute_strategy(self):
return self._strategy.execute()
# Uso del patrón Strategy
context = Context(ConcreteStrategy1())
result1 = context.execute_strategy()
print(result1) # Salida: "Estrategia concreta 1"
context.set_strategy(ConcreteStrategy2())
result2 = context.execute_strategy()
print(result2) # Salida: "Estrategia concreta 2"
